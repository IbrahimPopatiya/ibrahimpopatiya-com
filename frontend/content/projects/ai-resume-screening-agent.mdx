## Project Overview

Recruiters often need to manually read and evaluate hundreds of resumes
for a single role. This process is time‑consuming and can lead to
inconsistent candidate evaluation.

The AI Resume Screening & Feedback Agent was developed to automate the
initial screening stage of recruitment. The system reads resumes,
extracts key candidate information, evaluates candidate suitability for
roles, and assists recruiters by answering questions about applicants.

The platform acts as an AI recruitment assistant capable of
significantly reducing manual workload while improving consistency and
efficiency in hiring decisions.

## Key Features

-   **Resume Parsing & Information Extraction**: Automatically reads
    resumes and extracts candidate details such as skills, experience,
    and education.
-   **Candidate Profile Building**: Constructs structured candidate
    profiles from unstructured resume data.
-   **AI‑Powered Candidate Evaluation**: Uses LLM-based analysis to rank
    and evaluate candidates for specific roles.
-   **Recruiter Question Answering**: Recruiters can ask questions about
    candidates and receive AI-generated responses.
-   **Automated Feedback Email Generation**: Generates personalized
    candidate feedback emails automatically using AI.
-   **Modular AI Agent Architecture**: Designed with modular components
    to easily extend recruitment workflows.

## Technical Implementation

### Architecture

The system is designed using a modular AI agent architecture where each
component handles a specific task in the recruitment pipeline.

-   FastAPI backend for handling API requests
-   AI agent responsible for resume analysis and evaluation
-   Gradio interface for recruiter interaction
-   PostgreSQL database for storing candidate data and evaluations
-   Email integration for sending automated feedback

This architecture ensures scalability while allowing independent
development of each module.

### Frontend

The project uses Gradio to provide an interactive interface where
recruiters can upload resumes and interact with the AI agent.

``` typescript
// Example frontend interaction with API
async function analyzeResume(file) {
  const formData = new FormData();
  formData.append("resume", file);

  const response = await fetch("/api/resume/analyze", {
    method: "POST",
    body: formData
  });

  return response.json();
}
```

**Key Technologies:**

-   Interface: Gradio
-   API Communication: REST API
-   Interaction: Resume upload and recruiter Q&A interface

### AI/ML Components

The AI system analyzes candidate resumes and generates insights using

-   **Model Used**: OpenAI GPT models

-   **Prompt Engineering**: Structured prompts to evaluate candidate

## Challenges & Solutions

### Challenge 1: Extracting Structured Data from Unstructured Resumes

**Problem:** Resumes are often written in different formats and layouts,
making consistent data extraction difficult.

**Solution:** Implemented AI-based parsing using structured prompts and
resume preprocessing to reliably extract candidate information.

### Challenge 2: Candidate Evaluation Consistency

**Problem:** Human recruiters may evaluate candidates differently
depending on subjective judgment.

**Solution:** Developed AI evaluation prompts that analyze candidate
experience, skills, and education against job requirements to produce
consistent rankings.
